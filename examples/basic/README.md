# Basic Example

Deploy with a pre-existing GitHub App, passing its credentials directly.

Use this when you already have a GitHub App and want to provide its ID and
private key as Terraform variables. The webhook secret is auto-generated by the
module when not provided (see the `webhook_secret` output).

## Usage

```sh
terraform init
terraform apply \
  -var 'github_app_id=123456' \
  -var "github_app_private_key=$(cat path/to/private-key.pem)"
```

## Inputs

| Name                     | Description                          | Type     | Default       | Required |
| ------------------------ | ------------------------------------ | -------- | ------------- | :------: |
| `aws_region`             | AWS region to deploy resources into. | `string` | `"us-east-1"` |    no    |
| `github_app_id`          | GitHub App ID.                       | `string` | n/a           |   yes    |
| `github_app_private_key` | GitHub App private key (PEM format). | `string` | n/a           |   yes    |

## Outputs

| Name                   | Description                                      |
| ---------------------- | ------------------------------------------------ |
| `api_gateway_endpoint` | Base URL of the API Gateway.                     |
| `webhook_url`          | Webhook URL for the GitHub App settings.         |
| `sts_url`              | STS token exchange URL.                          |
| `webhook_secret`       | Webhook secret (auto-generated if not provided). |
| `healthz_url`          | Health check endpoint URL.                       |
